01 / Overview

What is MVP_2?

MVP_2 is a production-ready Android application that manages a product catalog with full user authentication, local caching, remote API integration, and enterprise analytics monitoring.

üîê

User Authentication

Complete register / login / logout lifecycle with JWT token management, "Remember Me" persistence, and secure local storage.

üì¶

Product Management

Full CRUD operations on products ‚Äî create, view, edit, delete ‚Äî with offline caching through Room and remote sync via REST API.

üìä

New Relic Analytics

Every user action, screen view, API call, and error is tracked via New Relic for real-time performance monitoring and observability.

Clean Architecture + MVVM

Strict separation into Presentation, Domain, and Data layers with ViewModel state management and single-source-of-truth data flow.

04 / Authentication Flow

Auth Flow Diagram

Complete sequence from app launch through registration, login, session restoration, and logout.

Login / Register Flow

1

App Launch

NavGraph checks TokenStore.hasToken(). If token exists ‚Üí navigate to products. Else ‚Üí navigate to auth.

2

Auth Screen Loaded

Single screen with toggle between Login and Register mode. All fields validated before submission.

3

User Submits Credentials

AuthViewModel calls LoginUseCase or RegisterUseCase ‚Üí AuthRepository ‚Üí EmailAuthManager ‚Üí POST /api/auth/login or POST /api/auth/register

4

Server Returns JWT Token

Response includes ServerAuthResponse with token, userId, firstName, lastName, email. Token saved to TokenStore.

5

New Relic ‚Äî Set User

NewRelicTracker.setUserAttributes(userId, email, name) called. All future events are associated with this user identity.

6

Login Success Screen

Splash screen greets user by name, then auto-navigates to ProductList. Auth screen removed from back stack.

7

Logout

LogoutUseCase clears TokenStore + SimplePrefs. Navigation pops back to auth screen. New Relic user attributes cleared.

Product Operations Flow

All product operations follow the same pattern: ViewModel ‚Üí UseCase ‚Üí Repository ‚Üí (Remote API + Local Cache) ‚Üí StateFlow update.

UI Update via StateFlow

06 / New Relic Analytics

Observability & Monitoring

Every meaningful user action and system event is tracked via the NewRelicTracker singleton, providing full visibility in the New Relic dashboard.

Event / Method When Triggered Attributes Captured
trackScreen(name) Every screen transition screenName, type: "navigation"
trackEvent("LoginSuccess") Successful login method: "email", userId
trackEvent("RegisterSuccess") Successful registration email, displayName
trackEvent("Logout") User taps logout userId
setUserAttributes() After login / register userId, email, name
trackEvent("ProductCreated") New product saved productName, price
trackEvent("ProductUpdated") Product edited uuid, updatedFields
trackEvent("ProductDeleted") Product removed uuid, productName
trackError(Throwable) Any caught exception error message, screen, context
trackMetric("LoadTime", ms) Performance milestones category: "Performance", value
startInteraction / endInteraction Long operations (API calls) interactionName, duration
incrementAttribute("api_calls") Every API request Running counter
noticeNetworkFailure() Network error caught url, method, startTime, endTime
New Relic Dashboard Access: Events appear under Custom Events in New Relic Insights. Use NRQL: SELECT * FROM ScreenView SINCE 1 day ago or SELECT count(*) FROM ProductCreated FACET productName for analytics queries.

08 / REST API Reference

API Endpoints

All API communication uses Retrofit with OkHttp. Bearer token is automatically injected by an auth interceptor for all protected endpoints.

Authentication Endpoints

POST /api/auth/register Create new account
POST /api/auth/login Login, get JWT token

Product Endpoints (Protected)
